# ethicallama service setup

Service units are generated by `ethllama setup`; no static unit in this
directory is installable. This avoids stale paths, user names, ports, and
credentials.

- `ethllama setup` installs a user service by default. It uses the current
  user, an absolute resolved executable, loopback configuration, and never
  invokes `sudo`.
- `ethllama setup --service-mode system` is explicit. Run it as the intended
  non-root user; it requires `sudo -n`, generates a PID-1 unit for that user,
  and provides root-owned configuration via `LoadCredential`.
- `ethllama setup --no-install` only writes user configuration. `--yes`
  preserves existing API-key authentication; use `--no-api-key` only to clear
  it deliberately.

The command reports linger status for user services but never changes it.
